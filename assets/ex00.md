# Exercício 00: BraiiiiiiinnnzzzZ

## Objetivo
Este exercício tem como objetivo demonstrar a diferença entre alocação de memória na **Stack** (Pilha) e no **Heap** (Monte) em C++, utilizando a criação de objetos da classe `Zombie`.

## Estrutura do Código

### 1. Classe `Zombie`
A classe representa um zumbi com um nome.
- **Construtor**: Inicializa o nome.
- **Destrutor**: Imprime uma mensagem debug quando o zumbi é destruído. Isso é crucial para visualizar o tempo de vida do objeto.
- **announce()**: O zumbi fala seu bordão.

### 2. Alocação na Stack - `randomChump`
```cpp
void randomChump(std::string name) {
    Zombie zombie(name); // Variável local
    zombie.announce();
} // Fim do escopo: 'zombie' é destruído automaticamente aqui
```
- **O que acontece**: O zumbi é criado como uma variável local dentro da função.
- **Vida útil**: Ele existe apenas enquanto a função `randomChump` está sendo executada. Assim que a função termina (fecha chaves `}`), o zumbi é automaticamente destruído.
- **Uso**: Ideal para objetos temporários que você só precisa usar dentro de um escopo específico e não precisa que existam depois.

### 3. Alocação no Heap - `newZombie`
```cpp
Zombie* newZombie(std::string name) {
    return new Zombie(name); // Alocação dinâmica com 'new'
}
```
- **O que acontece**: O zumbi é alocado dinamicamente na memória Heap usando a palavra-chave `new`. A função retorna um **ponteiro** para esse zumbi.
- **Vida útil**: O zumbi continua existindo na memória indefinidamente, mesmo após o fim da função `newZombie`, até que seja explicitamente deletado.
- **Uso**: Necessário quando você quer criar um objeto que sobreviva fora da função onde foi criado, ou quando lidar com grandes quantidades de dados que não cabem na Stack.
- **Cuidado**: **Você é responsável por destruir esse objeto!** Se não usar `delete`, ocorrerá um vazamento de memória (memory leak).

### 4. `main.cpp`
No arquivo principal, testamos ambos os casos:
1.  Chamamos `randomChump`: Vemos o zumbi nascer, falar e morrer imediatamente.
2.  Chamamos `newZombie`: Recebemos um ponteiro. O zumbi nasce e fala. Ele só morre quando chamamos `delete heapZombie` explicitamente no final.

## Em resumo
| Tipo | Onde | Palavra-chave | Vida útil | Limpeza |
|------|------|---------------|-----------|---------|
| **Stack** | `randomChump` | (Nenhuma) | Escopo da função | Automática (ao sair do escopo) |
| **Heap** | `newZombie` | `new` | Até ser deletado | Manual (usando `delete`) |
